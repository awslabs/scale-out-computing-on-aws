AWS DataSync is a secure, online service that automates and accelerates moving data between on-premises and AWS storage services.

Refer to AWS DataSync documentation to learn how to setup a secure, reliable, high‐speed file transfer between your current filers and EDH.

Your EDH environment must be using Amazon EFS to use AWS Transfer
AWS Transfer Family enables fully managed support for SFTP, FTPS and FTP to move large amounts of data into AWS.

Step 1: Configure AWS Transfer

  • Open AWS Console and navigate to the service named AWS Transfer Family then click Create Server
  • Select SFTP (SSH File Transfer Protocol) - file transfer over Secure Shell
  • Select Service Managed as identity provider
  • Choose Endpoint type depending on your setup. It's recommended to use VPC hosted for IP restrictions via your security groups. Publicly accessible means your AWS Transfer endpoint won't be protected by IP safelist.
  • Select Amazon EFS as Domain
  • Select Create a new role and use the latest TransferSecurityPolicy available
    • . Leave everything else as default.
  • In the Review and create section click Create server
Step 2: Create IAM role for your AWS Transfer Users
  • Open AWS Console and navigate to the service named IAM then click Roles on the left sidebar and finally click Create Role
  • Select AWS Service as Trusted Entity Type and select Transfer as Use Case
  • Select the AWS managed policy named AmazonElasticFileSystemClientReadWriteAccess
  • Select a Role name (for example TransferEFSClient and save it
Step 3: Download PEM Key File (Public)
  • Download your public SSH key
Step 4: Download your PEM key File (Private)

Click here to download it

Step 5: Register your AWS Transfer Users

You will need your user UID/GID. You can retrieve this value by typing id {user} on any EDH nodes (Linux). In the example below, the UID is 5001 and GID is also 5001

uid=5001({{user}}) gid=5001({{user}}) groups=5001({{user}})

Open AWS Console and navigate to the service named AWS Transfer Family select the server you have created and click Add User and enter the following information.

Username {{user}}
User ID The Posix UID of your user you have retrieved via `id` command
Group ID The Posix GID of your user you have retrieved via `id` command
Role The IAM role you have created (ex: TransferEFSClient)
Home Directory /data/home/{{ user }} (unless you have configured a different $HOME)
SSH Public Key The content of your SSH public key retrieved during Step 3
Step 6: Test

Open AWS Console and navigate to the service named AWS Transfer Family select the server you have created, and retrieve the Endpoint under Endpoint Details.

Your endpoint use the following syntax: s-{UNIQUE_ID}-.server.transfer.{AWS_REGION}.amazonaws.com

Connect to your AWS Transfer endpoint using your favorite FTP application via command line such as:

sftp -i "/PATH/TO/PRIVATE_KEY" {{user}}@{AWS_TRANSFER_ENDPOINT}

Alternatively, you can use WinSCP/FileZilla. Refer to the instructions available on this website and use your AWS Transfer endpoint as hostname.
